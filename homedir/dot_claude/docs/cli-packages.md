# CLI・パッケージ

不足しているCLIツールの扱いに関するルール。

## 不足しているCLIツール: ユーザーに確認する

CLIツールが現在の環境で利用できない場合、勝手にインストールしない。**必ずユーザーにインストール方法を確認する**。

### なぜ

- システムはchezmoiで管理されており、パッケージは宣言的に定義されている（darwinはHomebrew、debianはapt）。`brew install`・`npm -g`・`pip`・`cargo install` 等でアドホックにインストールすると、未追跡の状態が生まれ、宣言的管理と乖離する。
- どこに追加すべきか（恒久化するなら宣言リスト、一時利用なら別手段）はユーザーが決めることであり、エージェントが勝手に判断しない。

### 不足CLIに遭遇したとき

1. 自動でインストールしない。
2. 不足しているツールと、それが必要な理由をユーザーに伝える。
3. インストール方法（恒久的に宣言リストへ追加するか、一時的に入れるか等）をユーザーに確認する。

`brew`・`npm -g`・`pip`・`cargo install` 等を、ユーザーの明示的な指示なしに実行しない。
