# Feature Request Template

Use this template when the project has no `.github/ISSUE_TEMPLATE/feature*.md` or `.gitlab/issue_templates/feature*.md`.

```markdown
## Summary
[追加したい機能を1行で]

## Motivation
[この機能が必要な理由 — どんな問題を解決するか、誰のためか]

## Proposed Solution
[どのような機能であるべきか — UI、API、挙動の概要]

## Acceptance Criteria
- [ ] [完了条件1]
- [ ] [完了条件2]
- [ ] [完了条件3]

## Use Case
[具体的な使用シナリオ — 「ユーザーがXをしてYを得る」のような形式]

## Alternatives Considered
[他に検討した方法と、なぜこの方法を選んだか — なければセクションごと削除]

## Related Code
[既存コードで関連する箇所があれば記載 — コードベースから判明したもののみ]
- `path/to/related.ts`

## Out of Scope
[このIssueで扱わないこと — スコープを明確にするため]

## Additional Context
[モックアップ、参考実装、関連Issueなど]
```

## Section Rules

- **Motivation**: 「あったほうが良いから」では不十分。具体的な動機を書く。ユーザーから引き出す。
- **Proposed Solution**: 確定した設計でなくてよい。方向性が示されていればよい。実装詳細はPRで議論する。
- **Acceptance Criteria**: 完了の判定基準。テスト可能な形で書く。
- **Use Case**: 抽象的な機能説明だけでなく、具体的な使用例を含める。
- **Alternatives / Out of Scope / Additional Context**: 該当情報がない場合はセクションごと削除する。
