# Sprint Retrospective Template (KPT)

Generated by `retrospect-sprint`. Saved to `${paths.retros}/<sprint-id>.md`.

```markdown
# Sprint Retrospective: <sprint-id>

Period: <start-date> 〜 <end-date>
Capacity: <capacity>h
Investment ceiling: <ceiling>h
Completed: <completed>h (<percentage>%)
Velocity: <velocity>h

## Keep
[今回うまくいったこと、続けたいこと]

- 

## Problem
[今回うまくいかなかったこと、課題]

- 

## Try
[次のスプリントで試したいこと、改善案]

- 

## Notes
[追加のメモ、所感など。任意]
```

## Placeholders

| Key | Source |
|---|---|
| `<sprint-id>` | sprint metadata |
| `<start-date>`, `<end-date>` | sprint metadata |
| `<capacity>` | sprint metadata |
| `<ceiling>` | sprint metadata |
| `<completed>` | sum of estimates with Status=Done |
| `<percentage>` | `<completed>` / `<capacity>` × 100, rounded |
| `<velocity>` | same as `<completed>` for now |

## Section Rules

- **Keep / Problem / Try**: 各セクション最低1項目。空欄禁止。
- **Try は具体的に**: 「もっと頑張る」「気をつける」のような抽象は禁止。次スプリントで観測可能な変更を書く。
- **学びの活用**: Try の項目が再発パターンを示す場合の活用方針は `retrospect-sprint/SKILL.md` Step 7 を参照。retro ファイルや auto memory には書き込まない。
- **Notes**: 任意。
