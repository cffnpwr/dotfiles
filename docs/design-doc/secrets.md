# シークレット機構

リポジトリに含める秘密情報は[age](https://github.com/FiloSottile/age)で暗号化する。

## 暗号化方式

暗号化は[chezmoi](https://www.chezmoi.io/)の暗号化ファイル属性(`encrypted_`)を用い、ageの公開鍵(recipient)で暗号化したファイルとしてソースに格納する。
apply時に対応する秘密鍵(identity)で復号してホームへ配置する。
暗号化・復号はchezmoiの組み込み実装で完結し、外部のageバイナリを必要としない。

## 鍵の取り扱い

age鍵は次のように扱う。

- 秘密鍵(identity)はパスフレーズで暗号化して`.chezmoitemplates`に収録する。
- ブートストラップでパスフレーズを入力して復号し、`~/.config/chezmoi/key.txt`へ配置する。
- 公開鍵(recipient)は秘密でないため`chezmoi.toml`に平文で記載し、新規ファイルの暗号化に用いる。

復号は暗号化ファイルの適用に先立って行う。
通常はマシンごとに一度だけだが、鍵をローテーションした場合は収録した暗号鍵の差し替えに追従して再復号する。
実行順序はブートストラップの節で定める。

## ローテーション

鍵を交換する場合は次の順で行う。

1. 新しいage鍵を生成し、recipientとidentityを得る。
2. すべての暗号化ファイルを新しいrecipientで再暗号化する。
3. パスフレーズで暗号化した新しいidentityで`.chezmoitemplates`の収録鍵を差し替える。
4. `chezmoi.toml`のrecipientを更新する。
5. 変更をコミットする。

消費側の各環境では、次のapplyで収録鍵の変化が検知され、パスフレーズの再入力を経て新しい鍵が配置される。
鍵の配置は暗号化ファイルの適用より前に行われるため、1度のapplyで新しい鍵と再暗号化済みファイルが整合する。
