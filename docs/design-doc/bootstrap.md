# ブートストラップ・自動化

新規環境の構築と既存環境への変更適用を、[chezmoi](https://www.chezmoi.io/)のスクリプトで自動化する。
スクリプトの仕組みは[chezmoiの仕様](https://www.chezmoi.io/reference/target-types/#scripts)に従う。
本節では分類と実行順序の規約を定める。

## スクリプトの分類

操作の性質で3種に分ける。

- 一度きりでよい起点は`run_once`とする。
  鍵の復号が該当し、環境構築の起点として最初に一度だけ行う
- 存在を冪等に保つ操作は`run_`(毎回実行)とする。
  パッケージマネージャ・最小パッケージのブートストラップが該当する。
  毎回実行するが、存在を検査して満たされていなければ導入するため、定常時は安価で、欠落時のみ収束させる
- 宣言データに追従する操作は`run_onchange`とする。
  宣言に基づくパッケージのインストール、外部リポジトリの登録、ログインシェルの変更、システム設定、ログイン項目の登録、補完の生成などは、設定値・パッケージ一覧の変化に追従する必要がある。
  対象のデータをスクリプト本文へ展開し、その変化を検知して再適用する

## 実行順序

スクリプトは依存関係に沿った段階順で実行する。
番号は間隔を空けて付与し、段階の挿入に備える。
各段階の具体はOSで分岐し、該当しない段階は省かれる。

| 段階 | 種別 |
| --- | --- |
| 鍵の復号 | run_once |
| パッケージマネージャ・最小パッケージの導入 | run_(毎回) |
| 外部リポジトリの登録 | run_onchange |
| パッケージのインストール | run_onchange |
| ログインシェルの変更 | run_onchange |
| システム設定の適用 | run_onchange |
| ログイン項目の登録 | run_onchange |
| 補完の生成 | run_onchange |
