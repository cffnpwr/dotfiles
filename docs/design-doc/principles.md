# 設計原則

dotfiles全体に共通する設計上の原則を定める。
個々の設計判断はこれらの原則に従う。

## 単一情報源

全ての設定は[chezmoi](https://www.chezmoi.io/)ソースで一元管理する。
ホームディレクトリ上の設定ファイルは、ソースから生成される生成物である。
設定の変更はソースに対して行う。
chezmoi applyでホームディレクトリへ反映する。
ホームディレクトリを直接変更しても、chezmoi applyで上書きされて失われる。

## 冪等な再構築

環境構築は冪等な手続きとして表現する。
同じ手続きを何度実行しても安全に再実行でき、同じ構成へ収束する。
ここでの冪等性は手続きの再実行可能性を指す。
各ツールのバージョンを特定時点へ固定することは含まない。
バージョン固定の要否は、導入手段ごとに判断する。

## OS別ネイティブなパッケージ導入

パッケージは抽象化レイヤを挟まず、各OSがネイティブに備える手段で導入する。
手段の統一よりも、各OSでの素直さと長期的な保守性を優先する。

## シークレットの暗号化

リポジトリに含める秘密情報は暗号化し、平文では置かない。
復号はchezmoi applyの実行時に行う。
